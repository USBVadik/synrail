# Baseline Comparison Candidate 001

## Purpose

Select the first concrete scenario for head-to-head comparison between `Synrail` and the simpler baseline.

This document exists so the first comparison uses a scenario that is already evidence-rich instead of a fresh speculative case.

## Selected candidate

The first comparison candidate is:

- the narrow exact-task router/trigger bugfix class already exercised in the governed exact-retry path

More specifically:

- a small routing fix with plausible false-success risk
- one meaningful closure decision
- one patch-bearing file
- one history of real blocker narrowing before accepted closure

## Why this candidate is first

This candidate is the best first comparison target because it already has:

- a real killer-path fit
- real artifact-bearing exact runs
- real false-success pressure
- real closure-grade outcomes
- enough surrounding history to reconstruct what a simpler operator loop would likely have looked like

It is a stronger first comparison candidate than:

- a fresh incident with no proof trail yet
- a broad downstream capability issue
- a regression that never enters exact-task semantics

## What the baseline should replay

The lightweight baseline replay should try to answer:

- if the same narrow task had been run with a bounded task statement, one agent attempt, and one manual sanity check, what would likely have happened?

The replay should remain disciplined and modest.

It should not invent impossible certainty.

It should compare:

- likely false-success exposure
- likely blocker clarity
- likely recovery cost

## Why this is fair enough

This candidate is not “clean lab science.”

It is still good enough for a first comparison because:

- the scenario is narrow
- the comparison target is concrete
- the costs of `Synrail` are visible
- the risks of baseline hand-waving are also visible

The goal is not perfect benchmarking.

The goal is an honest first product reading.

## What to avoid

Do not overclaim from this candidate:

- it will not prove all of `Synrail`
- it will not settle broad generality
- it will not justify shell growth by itself

It only needs to answer:

- does `Synrail` look materially better than the simpler baseline on its own best current path?

## Next move

Use this candidate to produce:

- one compact comparison record

That record should end with one of three verdicts:

- `Synrail better`
- `baseline good enough`
- `unclear`
