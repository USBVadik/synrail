## Перепроверка CRITICAL findings аудита #1

### CRITICAL #1: Verification commands never re-executed

**Статус: ОТКРЫТО (стало хуже)**

На main не только не реализована re-execution — удалены даже поля `verification_command` и `verification_result` из bundle. Весь NLP-движок удалён (37 vacuous фраз, 130 глагольных форм, 70% word-overlap). Вместо этого:
- readback: 6 keywords + `len >= 24`
- scenario: 7 keywords + `len >= 24`

Это не упрощение ради ясности — это демонтаж proof guard. Любая строка ≥24 символа проходит semantic check.

### CRITICAL #2: Doctor gates — boolean bypass

**Статус: ОТКРЫТО**

Все 5 флагов (`--clean-surface`, `--artifact-viable`, `--helper-ok`, `--credentials-ok`, `--prompt-identity-ok`) по-прежнему дают безусловный PASS без маркировки override в записи. Единственное изменение: добавлена проверка на пустой target_identity (строки 151-152 doctor_v1.py).

### CRITICAL #3: Closure gaming via fabricated JSON

**Статус: ОТКРЫТО (минимальное улучшение)**

Добавлена проверка `run_id_mismatch` (строки 66-68 closure_v0.py). Но:
- Атакующий может задать одинаковый run_id в state и bundle
- По-прежнему нет подписей, хешей, tamper detection
- Все 11 проверок — dict-value сравнения

## Новые уязвимости (следующий слой)

### NEW-1: TOCTOU Race Condition (MEDIUM)
**Файл**: synrail_doctor_v1.py, строки 232, 262, 274, 334, 362

`Path.exists()` → промежуток → `subprocess.run()` с тем же путём. Между проверкой и использованием файл может быть подменён через symlink.

### NEW-2: Symlink traversal в credential probe (MEDIUM)
**Файл**: synrail_doctor_v1.py:333

`Path(value).expanduser()` без `.resolve()`. Симлинк на `/proc/self/environ` пройдёт проверку "credential file exists".

### NEW-3: Path traversal через --*-file аргументы (MEDIUM)
**Файлы**: closure_v0, bundle_v0, doctor_v1 и др.

Все `--state-file`, `--final-result` и подобные аргументы принимаются без валидации scope. `../../etc/passwd` — допустимый путь.

### NEW-4: JSON parsing без schema validation (LOW-MEDIUM)
**Файл**: synrail_alpha_telemetry_v0.py:113

`json.loads(line)` без schema проверки. При контроле над telemetry log — injection произвольных структур.

### NEW-5: Semantic proof guard regression (HIGH)
**Файл**: synrail_bundle_v0.py

Readback проходит при ЛЮБОМ тексте ≥24 символа ИЛИ содержащем одно из 6 ключевых слов. Scenario — аналогично. В аудите #1 были 37 vacuous фраз + 130 глагольных форм + word-overlap detection. Всё удалено. Порог входа для фальшивого proof снижен с "обойти NLP-движок" до "написать 24 символа".

### NEW-6: Удвоение JSON IO дублирования (LOW-MEDIUM)
**Файлы**: 51 файл в tools/reference/

102 дублированных функции load_json/save_json (было 48). Расхождение уже начинается: checkpoint добавляет mkdir, telemetry добавляет load_json_if_exists, artifact_consistency добавляет load_json_safe. Три варианта с разной семантикой ошибок.

## Вердикт

### Что произошло с прошлого аудита

1. **Hardening был написан, но не доставлен.** На feature branch реализованы T0.1, T0.2, T0.3, T1.1, T1.2. На main — ничего. Это классическая ситуация "работа сделана, но не shipped".

2. **Main получил clean-room extraction вместо hardening.** Merge pre-alpha (17ed555) привёл в main минимальную кодовую базу с 2072 файлами (+192k строк), но **без** hardening-коммитов. Последующие PR добавили только deploy guard и minor fixes.

3. **Тестовая guard surface сократилась вдвое.** 218 из 492 тестов удалены. Особенно критично: 69 adversarial тестов из truth_regressions, весь unseen_validation_pack (21 тест), весь claim_validation_pack (8 тестов).

4. **NLP proof guard демонтирован.** Вместо усиления — замена на `len >= 24` + 6 keywords. Это объективное ослабление, а не "упрощение".

### Общий вердикт

**Synrail на main отступил от позиции аудита #1.** Кодовая база стала больше, но trust surface стала тоньше. Все 3 CRITICAL findings остаются открытыми. Guard surface ослабла. Единственные реальные улучшения на main: atomic restore (T2.1), drift cleanup (T5.0), частичный CLI split (T5.4), deploy guard.

Hardening-код существует на feature branch, но разрыв между ветками — это не "в процессе", а "не доставлено". Production = main.
