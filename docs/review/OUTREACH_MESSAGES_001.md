# Outreach Messages 001

Use these as send-ready templates.

## 1. Senior Engineer

```text
I’m testing a narrow product called Synrail.

It’s not a platform. The wedge is smaller: stop coding-agent false green, keep one verified fallback, and make non-green repair/restore paths explicit.

I want blunt technical criticism, especially on:
- truth rigor
- false accept / false reject risk
- whether the shell creates leverage or just justified ceremony

If you’re open, I’ll send a small alpha pack that takes about 10 minutes to attack and a deeper technical map if you want to go further.
```

## 2. Product-Minded Critic

```text
I’m pressure-testing a very narrow product idea called Synrail.

The idea is not “better AI orchestration.” It’s “don’t let agent work get accepted too early, and make the non-green path cheaper and clearer.”

I need a hard product read on two things:
- does this create obvious workflow leverage fast enough?
- or does it still feel like process overhead dressed up as rigor?

If you’re willing, I’ll send one small alpha pack and ask for a blunt reaction.
```

## 3. Infra / Reliability Critic

```text
I’m testing Synrail, a proof-first control layer for coding-agent execution.

The narrow claim is that it should be better than baseline when false green is expensive, restore matters, and handoff/re-entry needs to stay honest.

What I need from you is not polish feedback. I need teardown feedback on:
- where the trust anchors still look self-issued
- where bounded readiness or proof claims are still too strong
- whether restore/re-entry actually buys anything in practice

I have a small alpha pack and a code/technical map if you’re open to reviewing it.
```

## 4. Follow-Up After They Say Yes

```text
Perfect. Here’s the smallest useful entry:

1. docs/core/ALPHA_TEST_PACK_001.md
2. docs/review/EXTERNAL_CRITIQUE_PACK_001.md
3. docs/review/KNOWN_WEAKNESSES_001.md

If you only have 10-15 minutes, just run the quickstart and tell me:
- what felt useful
- what felt ceremonial
- what Synrail actually saved you from, if anything

If you want the deeper map, I’ll point you to the technical files too.
```

## 5. Follow-Up If They Go Cold

```text
Quick nudge on Synrail.

No pressure if the timing is bad, but if you do look at it, the most useful thing you could tell me is not “is this cool?”

It’s:
- where does it still feel fake?
- where does the shell still feel like ceremony?
- and what part, if any, would survive if the product were cut in half?
```
